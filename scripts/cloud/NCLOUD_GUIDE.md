## STEP 0. 준비물 확인
> 지금 뭘 하는 건지: 학습을 시작하기 전에 필요한 것들이 다 있는지 확인해요.

아래 4가지가 모두 있어야 시작할 수 있어요. 하나씩 체크해 보세요.

| ✅ | 준비물 | 설명 |
|----|--------|------|
| ☐ | **네이버클라우드 계정 (기업회원)** | 개인 계정이 아닌 기업 계정이어야 해요. 없으면 www.ncloud.com 에서 가입하세요. |
| ☐ | **외장 SSD (삼성 T7 Shield 1TB)** | AI 모델 파일이 크기 때문에 외장 SSD가 필요해요. 맥북에 미리 꽂아두세요. |
| ☐ | **맥북 (터미널 앱)** | 맥북의 "응용 프로그램 → 유틸리티 → 터미널" 앱을 써요. |
| ☐ | **신용카드** | 서버 요금이 카드로 결제돼요. 네이버클라우드 계정에 카드를 등록해야 해요. |

> 💡 **신용카드 등록하는 법**: 네이버클라우드 콘솔 로그인 → 오른쪽 위 내 정보 → 결제 수단 → 카드 등록

## STEP 1. GPU 서버 만들기
> 지금 뭘 하는 건지: AI 학습에 필요한 강력한 컴퓨터(GPU 서버)를 클라우드에서 빌려요.

### 1-1. 콘솔 접속하기

1. 맥북에서 사파리나 크롬을 열어요.
2. 주소창에 **www.ncloud.com** 을 입력하고 Enter를 눌러요.
3. 오른쪽 위 **[로그인]** 버튼을 클릭하고, 기업 계정으로 로그인해요.
4. 로그인 후 오른쪽 위 **[콘솔]** 버튼을 클릭해요.
5. 검은색 또는 파란색 대시보드 화면이 나오면 성공이에요.

### 1-2. 서버 만들기 시작

1. 왼쪽 메뉴에서 **[Services]** 를 클릭해요.
2. 아래로 스크롤해서 **[Compute]** 를 찾아 클릭해요.
3. 그 안에 **[Server]** 를 클릭해요.
4. 오른쪽 위 파란 버튼 **[서버 생성]** 을 클릭해요.

### 1-3. 서버 설정 (중요! 아래 그대로 선택하세요)

**① 서버 이미지 선택**
- **Ubuntu** 탭을 클릭해요.
- **Ubuntu Server 20.04 (64-bit)** 를 선택해요.
- 오른쪽 아래 **[다음]** 을 클릭해요.

**② 서버 타입 선택**
- 왼쪽 필터에서 **[GPU]** 탭을 클릭해요.
- **V100 32GB** 타입을 찾아서 선택해요.
  - 이름에 `v1.g2` 또는 `V100` 이 들어있는 것을 고르면 돼요.
- **서버 이름** 칸에 `butler-training` 이라고 입력해요.
- **스토리지** 는 **100GB SSD** 이상으로 선택해요.
- **[다음]** 을 클릭해요.

> 💡 **V100 32GB를 추천하는 이유**: Qwen2.5-7B 모델을 4bit로 압축해서 학습하는데, 32GB가 있으면 안정적으로 돌아가요.

**③ 인증 키 만들기 (SSH 키)**

인증 키는 서버의 자물쇠 열쇠예요. 만들어서 꼭 저장해야 해요!

1. **[인증 키 생성]** 을 선택해요.
2. 키 이름 칸에 `butler-key` 라고 입력해요.
3. **[생성 및 저장]** 버튼을 클릭해요.
4. `butler-key.pem` 파일이 맥북 다운로드 폴더에 자동 저장돼요.
5. 이 파일을 **절대로 잃어버리면 안 돼요!** 없으면 서버에 접속할 수 없어요.
6. **[다음]** 을 클릭해요.

**④ 최종 확인 후 생성**
- 설정 내용이 맞는지 확인해요.
- **[서버 생성]** 버튼을 클릭해요.
- 서버 만드는 데 2~5분 걸려요. 기다려요!

### 1-4. 공인 IP 신청하기 (외부 접속 주소)

서버를 만들었어도 공인 IP가 없으면 접속할 수 없어요.

1. 왼쪽 메뉴 **[Compute]** → **[Public IP]** 를 클릭해요.
2. 오른쪽 위 **[공인 IP 신청]** 을 클릭해요.
3. 팝업이 나오면 그대로 **[확인]** 을 클릭해요.
4. 새로 생긴 IP 주소(숫자)를 클릭해요.
5. **[서버 적용]** 을 클릭하고, `butler-training` 서버를 선택해요.
6. **[적용]** 을 클릭해요.
7. 서버 목록으로 돌아가면 공인 IP 칸에 숫자(`223.xxx.xxx.xxx`)가 표시돼요.
8. 이 숫자가 **서버 주소**예요. 메모장에 적어두세요!

> ⚠️ **중요 경고**: GPU 서버는 **정지(Stop)** 상태에서도 요금이 나와요!
> 학습이 끝나면 반드시 **삭제(Delete)** 해야 해요. STEP 8을 꼭 읽어주세요.

## STEP 2. 서버에 접속하기 (SSH)
> 지금 뭘 하는 건지: 맥북 터미널에서 클라우드 서버로 원격 접속해요.

### 2-1. 터미널 열기

1. 맥북에서 **Spotlight 검색** (Command + Space) 을 눌러요.
2. `터미널` 이라고 입력하고 Enter를 눌러요.
3. 검은색(또는 흰색) 화면이 열리면 준비 완료예요.

### 2-2. 인증 키 파일 준비

터미널에 아래 명령어를 한 줄씩 입력하고 Enter를 눌러요:

```bash
# 인증 키 파일이 있는 위치로 이동
cd ~/Downloads

# 키 파일의 권한을 설정 (보안을 위해 꼭 해야 해요)
chmod 400 butler-key.pem
```

> 💡 `chmod 400` 은 "이 파일은 나만 읽을 수 있다"는 설정이에요. 이걸 안 하면 SSH 접속이 거부돼요.

### 2-3. SSH로 서버 접속하기

아래 명령어에서 `서버IP주소` 부분을 STEP 1에서 메모한 숫자로 바꿔요:

```bash
ssh -i ~/Downloads/butler-key.pem root@서버IP주소
```

예시 (실제 IP로 바꾸세요):
```bash
ssh -i ~/Downloads/butler-key.pem root@223.130.xxx.xxx
```

### 2-4. 처음 접속 시 나오는 질문

처음 접속하면 이런 메시지가 나와요:

```
The authenticity of host '223.xxx.xxx.xxx' can't be established.
Are you sure you want to continue connecting (yes/no)?
```

`yes` 라고 입력하고 Enter를 눌러요.

> 💡 **왜 "yes"를 입력하나요?**
> 이 컴퓨터(서버)를 처음 만나는 거라서 "이 서버를 믿을 수 있어요?"라고 물어보는 거예요.
> 우리가 방금 만든 서버니까 "네, 믿어요"라고 대답하는 거예요.
> 한 번 "yes"를 입력하면 다음부터는 안 물어봐요.

### 2-5. 접속 성공 확인

아래처럼 화면이 바뀌면 접속 성공이에요! 🎉

```
Welcome to Ubuntu 20.04.6 LTS (GNU/Linux 5.4.0 ...)

root@butler-training:~#
```

`root@butler-training:~#` 이 보이면 이제 서버 안에서 명령어를 입력하는 거예요.

## STEP 5. 데이터 수집 (AI-18)
> 지금 뭘 하는 건지: AI가 공부할 한국어 책과 문서를 인터넷에서 가져와요.

AI는 학습 데이터를 먹고 똑똑해져요. 사람이 책을 읽어서 공부하는 것처럼,
AI도 많은 한국어 텍스트를 읽어야 해요. 이 단계에서 그 "교과서"를 준비해요.

### 5-1. 데이터 수집 스크립트 실행

```bash
bash scripts/cloud/collect_data.sh
```

### 5-2. 무슨 일이 일어나나요?

이 스크립트는 아래 작업을 자동으로 해요:

1. **한국어 공개 데이터 다운로드** — 공개된 한국어 문서와 대화 데이터를 가져와요
2. **데이터 정리** — 가져온 문서를 AI가 읽을 수 있는 형태로 변환해요
3. **저장** — `data/synthetic_v40/train.jsonl` 파일로 저장해요

### 5-3. 완료 시간

데이터 생성 방식에 따라 보통 **1~5분** 정도 걸려요.

```

   검증 데이터: data/synthetic_v40/validation.jsonl
```

## STEP 6. AI 학습 실행 (Phase B)
> 지금 뭘 하는 건지: AI가 모은 데이터를 읽으면서 공부(학습)를 해요. 시간이 제일 오래 걸려요.

### 6-1. 새 screen 세션 시작

```bash
screen -S training
```

> 💡 왜 screen을 또 쓰나요? 학습은 2~4시간이 걸려요.
> 그동안 맥북을 닫거나 인터넷이 끊겨도 학습이 계속되게 하려고요.

### 6-2. 학습 시작

```bash
bash scripts/cloud/run_training.sh
```

### 6-3. 학습이 잘 되고 있는지 확인하기

화면에 이런 숫자들이 흘러가면 정상이에요:

```
=== QLoRA 파인튜닝 시작 ===
trainable params: 39,976,960 || all params: 7,721,795,584
{'loss': 2.4532, 'learning_rate': 0.0002, 'epoch': 0.12}
{'loss': 2.1847, 'learning_rate': 0.00019, 'epoch': 0.24}
```

- `loss` 숫자가 점점 작아지면 학습이 잘 되고 있는 거예요! 📉
- `epoch` 숫자가 올라가면서 3.0이 되면 학습이 끝나요.

### 6-4. 예상 학습 시간

| GPU 종류 | 예상 시간 |
|----------|----------|
| V100 32GB (추천) | 약 2~4시간 |
| A100 40GB | 약 1~2시간 |

### 6-5. 터미널 창을 닫아야 할 때 (screen 숨기기)

키보드에서 **Ctrl + A** 를 누른 다음 **D** 를 눌러요 (동시에 누르는 게 아니에요, 순서대로요).

그러면 이런 메시지가 나와요:
```
[detached from XXXXX.training]
```
이제 터미널 창을 닫거나 맥북을 닫아도 학습이 계속 돌아가요!

### 6-6. 나중에 학습 상황 다시 보기

나중에 서버에 다시 SSH 접속한 뒤:

```bash
# 실행 중인 screen 목록 확인
screen -ls

# training 세션으로 돌아가기
screen -r training
```

### 6-7. 학습 완료 메시지

학습이 끝나면 이런 메시지가 나와요:
```
 학습완료! 이제 파일을 다운로드하세요 🎉
 다음 단계: bash scripts/cloud/download_and_cleanup.sh
```

## STEP 7. 결과물 다운로드
> 지금 뭘 하는 건지: 학습된 AI 모델 파일을 압축해서 맥북과 외장SSD에 저장해요.

### 7-1. 모델 파일 압축

서버에서 아래 명령어를 실행해요:

```bash
bash scripts/cloud/download_and_cleanup.sh
```

완료되면 서버에 `butler_model_v1.tar.gz` 파일이 만들어져요.

### 7-2. 맥북으로 파일 다운로드 (scp)

**맥북의 새 터미널 창을 열고** (서버 접속 상태가 아닌 맥북 터미널) 아래를 입력해요:

```bash
# 홈 폴더로 이동
cd ~

# 서버에서 맥북으로 파일 다운로드 (서버IP주소를 실제 IP로 바꾸세요)
scp -i ~/Downloads/butler-key.pem \
    root@서버IP주소:~/tristan050-ai_ondevice_APP/butler_model_v1.tar.gz \
    ~/Downloads/

# 체크섬 파일도 다운로드
scp -i ~/Downloads/butler-key.pem \
    root@서버IP주소:~/tristan050-ai_ondevice_APP/butler_model_v1.tar.gz.sha256 \
    ~/Downloads/
```

다운로드가 끝나면 진행률이 100%가 되고 커서가 다시 나타나요.

### 7-3. 다운로드 완료 확인

```bash
# 파일이 제대로 받아졌는지 확인 (체크섬 비교)
cd ~/Downloads
shasum -a 256 butler_model_v1.tar.gz
cat butler_model_v1.tar.gz.sha256
```

두 줄에서 맨 앞의 긴 숫자(해시)가 **같으면** 파일이 완전히 받아진 거예요. ✅

### 7-4. 외장 SSD에 복사하기

```bash
# 외장SSD가 연결된 경로 확인 (Volumes 아래에 표시됨)
ls /Volumes/

# 외장SSD로 복사 (SSD이름을 실제 이름으로 바꾸세요, 예: T7Shield)
cp ~/Downloads/butler_model_v1.tar.gz /Volumes/T7Shield/
cp ~/Downloads/butler_model_v1.tar.gz.sha256 /Volumes/T7Shield/

# 복사 확인
ls -lh /Volumes/T7Shield/butler_model_v1.tar.gz
```

파일 크기가 표시되면 복사 완료예요!

> # ⚠️ 경고 1: 서버를 삭제하지 않으면 요금이 계속 나와요!
>
> GPU 서버는 **1시간에 약 1,300~2,000원**이 청구됩니다.
> 하루만 켜둬도 **3만 원 이상**, 한 달이면 **100만 원 이상**이 나올 수 있어요.
> 파일을 다 받은 즉시 아래 단계를 따라 서버를 **삭제**해 주세요.

---

> # ⚠️ 경고 2: 정지(Stop)가 아니라 삭제(Delete)를 해야 해요!
>
> **정지(Stop)** = 서버를 잠깐 꺼두는 것 → **요금 계속 발생** ❌
>
> **삭제(Delete)** = 서버를 완전히 없애는 것 → **요금 중단** ✅
>
> 반드시 **삭제**를 선택해야 해요!

---

### 8-1. 공인 IP 먼저 반납하기

공인 IP를 반납하지 않으면 IP 사용 요금이 따로 청구돼요.

1. 네이버클라우드 콘솔에서 **[Compute]** → **[Public IP]** 를 클릭해요.
2. `butler-training` 서버에 연결된 IP 주소 왼쪽 체크박스를 클릭해요.
3. 상단 버튼 **[서버 적용 해제]** 를 클릭해요 → 팝업에서 **[확인]** 을 클릭해요.
4. 다시 해당 IP 체크박스를 클릭해요.
5. **[공인 IP 반납]** 을 클릭해요 → 팝업에서 **[확인]** 을 클릭해요.
6. IP 목록이 비어있으면 완료예요. ✅

### 8-2. 서버 종료하기

1. 왼쪽 메뉴 **[Compute]** → **[Server]** 를 클릭해요.
2. `butler-training` 서버의 왼쪽 체크박스를 클릭해요.
3. 상단 **[서버 관리 및 설정 변경]** 버튼을 클릭해요.
4. **[서버 종료]** 를 선택해요.
5. 팝업에서 **[확인]** 을 클릭해요.
6. 서버 상태가 **"정지"** 로 바뀔 때까지 기다려요 (1~3분).

### 8-3. 서버 삭제하기

1. 상태가 "정지"가 된 서버 체크박스를 다시 클릭해요.
2. **[서버 관리 및 설정 변경]** → **[서버 삭제]** 를 클릭해요.
3. 팝업에서 서버 이름 `butler-training` 을 확인하고 **[확인]** 을 클릭해요.

### 8-4. 삭제 완료 확인

서버 목록 페이지에서 `butler-training` 이 **사라지면** 삭제 완료예요!

---

> # ⚠️ 경고 3: 서버 목록이 비어있어야 완전히 안전해요!
>
> 서버 목록 페이지를 새로 고침(Command + R)해서
> `butler-training` 이 목록에 없는지 꼭 다시 확인해 주세요.
>
> ✅ 목록이 비어있거나 butler-training이 없으면 안전합니다.
>
> ❌ 아직 목록에 있다면 삭제가 완료되지 않은 거예요. 8-2, 8-3 단계를 다시 확인해 주세요.

### Q1. screen 세션으로 다시 접속하는 법

```bash
# 현재 실행 중인 screen 목록 보기
screen -ls

screen -r training   # training 세션
screen -r setup      # setup 세션

# 세션이 여러 개일 때는 번호로 접속
screen -r 12345.training
```

---

### Q2. 학습 중 에러 확인하는 법

학습 중에 에러가 났을 때 확인하는 방법이에요:

```bash
# 학습 로그 파일에서 마지막 50줄 보기
tail -50 output/training.log

# 에러(ERROR, Error) 키워드로 검색
grep -i "error" output/training.log | tail -20

# 실시간으로 로그 보기
tail -f output/training.log
```

에러 메시지가 보이면 해당 내용을 복사해서 팀에 공유해 주세요.

---

### Q3. 서버 접속이 안 될 때

**"Connection refused" 또는 "Connection timed out" 에러**

1. 네이버클라우드 콘솔에서 서버가 **"운영중"** 상태인지 확인해요.
2. 공인 IP가 서버에 연결되어 있는지 확인해요.
3. 아래 명령어로 다시 시도해요:

```bash
ssh -i ~/Downloads/butler-key.pem -o ConnectTimeout=30 root@서버IP주소
```

**"WARNING: UNPROTECTED PRIVATE KEY FILE!" 에러**

```bash
chmod 400 ~/Downloads/butler-key.pem
```

---

### Q4. 요금 확인하는 법

1. 네이버클라우드 콘솔 로그인해요.
2. 오른쪽 위 내 프로필 → **[마이페이지]** 를 클릭해요.
3. **[비용 관리]** → **[이용 현황]** 을 클릭해요.
4. 현재까지 발생한 요금을 확인할 수 있어요.

> 💡 서버를 삭제했는데도 요금이 나온다면, 공인 IP 반납을 못 했거나 다른 서버가 켜져 있을 수 있어요. 모든 서버와 공인 IP 목록을 확인해 보세요.
