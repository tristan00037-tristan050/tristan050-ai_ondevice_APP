## 1. Butler 설치 (macOS)

### 1.1 .dmg 다운로드

배포 채널로 전달받은 `Butler_0.1.0_aarch64.dmg` 파일을 다운로드합니다.

### 1.2 Applications 폴더로 드래그

1. `Butler_0.1.0_aarch64.dmg` 더블클릭
2. 마운트된 창에서 `Butler.app`을 `Applications` 폴더로 드래그
3. 복사 완료 후 DMG 창 닫기 (디스크 이미지 꺼내기)

### 1.3 첫 실행 시 보안 경고 우회

Butler는 Apple Developer 프로그램에 미가입 상태(코드 서명 없음)입니다.  
첫 실행 시 macOS Gatekeeper가 경고를 표시합니다.

**방법 A (권장) — 우클릭으로 열기:**
1. Finder에서 `Applications` → `Butler.app` 찾기
2. Butler.app **우클릭** → **열기** 선택
3. "확인되지 않은 개발자" 경고 팝업에서 **열기** 클릭
4. 이후 실행부터는 정상적으로 더블클릭 가능

**방법 B — 시스템 설정 경유:**
1. Butler.app 더블클릭 → 경고 팝업 확인(닫기)
2. `시스템 설정` → `개인 정보 보호 및 보안` → 화면 하단
3. "Butler"가 차단됐다는 메시지 옆 **"확인 없이 열기"** 클릭
4. Touch ID 또는 비밀번호 입력

> **주의:** 이 절차는 사내 배포 버전에만 해당합니다. 출처 불명 파일에는 동일한 우회를 시도하지 마세요.

### 1.4 Python 의존성 설치

Butler의 AI 엔진은 Python 사이드카(butler_sidecar.py)로 구동됩니다.  
배포 패키지에 포함되어 있으나, 최초 1회 아래 명령을 실행해야 합니다.

```bash
# 터미널에서 실행 (Butler 배포 폴더로 이동 후)
pip3 install -r requirements-serving.txt
```

> Python 3.10 이상이 필요합니다. `python3 --version`으로 확인하세요.

> **⚠️ LLM 실제 응답을 받으려면 llama-cpp-python이 필요합니다.**  
> `requirements-serving.txt`에 포함되어 있지만, Apple Silicon Mac에서는  
> Metal GPU 가속을 위해 아래 명령으로 별도 설치를 권장합니다:
>
> ```bash
> # Apple Silicon (Metal GPU 가속)
> CMAKE_ARGS="-DGGML_METAL=ON" pip3 install llama-cpp-python
> ```
>
> 미설치 시에도 Butler는 작동하지만 "stub 응답"이 표시됩니다.

### 1.5 AI 모델 다운로드 및 경로 설정

Butler는 로컬 GGUF 모델 파일을 사용합니다. 모델 파일은 앱과 별도로 관리합니다.

**모델 다운로드 (권장: Qwen3-4B Q4_K_M):**

```bash
# Hugging Face CLI 사용
pip3 install huggingface_hub
huggingface-cli download Qwen/Qwen3-4B-GGUF qwen3-4b-q4_k_m.gguf \
  --local-dir ~/butler-models
```

**모델 경로 설정:**

```bash
# ~/.zshrc 또는 ~/.bash_profile에 추가
export BUTLER_MODEL_PATH="$HOME/butler-models/qwen3-4b-q4_k_m.gguf"
```

설정 후 새 터미널을 열거나 `source ~/.zshrc`를 실행하세요.

**LLM 추론 라이브러리 설치 (선택 — 고품질 응답을 원할 경우):**

```bash
# Apple Silicon Mac
CMAKE_ARGS="-DGGML_METAL=ON" pip3 install llama-cpp-python

# Intel Mac
pip3 install llama-cpp-python
```

> `llama-cpp-python` 미설치 시에도 Butler는 작동합니다. 이 경우 "stub 응답"이 표시되며,  
> 상단 노란색 배너에서 설치 안내를 확인할 수 있습니다.

## 2. 첫 실행

### 2.1 모델 로딩 대기

Butler를 처음 실행하면 로컬 AI 모델(Qwen3-4B, Q4_K_M 양자화)을 로드합니다.

- **예상 시간:** 1~3분 (MacBook Air M1 기준)
- 로딩 중 진행 표시줄이 나타납니다
- 인터넷 연결이 없어도 정상 작동합니다

### 2.2 (선택) Personal Pack 인덱싱

회사 문서를 AI가 참조하도록 인덱싱합니다.

1. 설정 → Personal Pack → 폴더 선택
2. 문서 5~50건 권장 (많을수록 응답 품질 향상)
3. 인덱싱 시간: 100건당 약 2~5분

### 2.3 Wi-Fi OFF 테스트 (외부 송신 0 검증)

Butler가 인터넷 없이 작동하는지 확인하세요:

1. Wi-Fi 끄기 (메뉴 바 → Wi-Fi OFF)
2. Butler에서 아무 카드나 선택 → 텍스트 입력 → 전송
3. 정상 응답 수신 확인
4. 상단 🔒 배지가 "Local-only Mode" 표시 유지 확인

## 3. 카드 6개 사용법

화면 상단 6개 카드 중 하나를 선택해 작업 모드를 지정합니다.  
카드 선택 없이 자유 입력도 가능합니다.

### 3.1 📥 요청 핵심 파악·정리

**용도:** 받은 메일·메시지에서 핵심 요청 추출  
**입력 예시:**
```
[이메일 내용 붙여넣기]
이 메일에서 무엇을 언제까지 어떤 자료로 처리해야 하는지 정리해줘.
```
**출력 예시:** "요청: 제안서 수정 / 기한: 5/15 / 필요 자료: 작년 실적 파일"

### 3.2 🔄 남의 문서 → 우리 양식

**용도:** 외부에서 받은 문서를 우리 회사 양식으로 변환  
**입력 예시:**
```
[외부 문서 첨부] + [우리 양식 파일 첨부]
이 외부 제안서를 우리 표준 보고서 양식에 맞게 변환해줘.
```

### 3.3 📄 기존 문서 기반 새 초안

**용도:** 과거 작성 문서를 참고해 새 초안 생성  
**입력 예시:**
```
[과거 보고서 첨부]
이 형식 참고해서 이번 달 마케팅 성과 보고서 초안 써줘.
핵심 지표: 전환율 3.2%, 신규 유입 1,200명
```

### 3.4 📝 첨부 문서 수정·보완

**용도:** 작성 중인 문서의 빠진 내용·수정안 제안  
**입력 예시:**
```
[작성 중 문서 첨부]
이 기획서에 빠진 부분이 있는지 확인하고, 리스크 분석 섹션을 추가해줘.
```

### 3.5 💰 통장·거래내역 → 회계 분류

**용도:** 통장 거래내역 자동 계정과목 분류  
**지원 형식:** CSV, PDF, XLSX (거래내역 형식)  
**입력 예시:**
```
[거래내역 파일 첨부]
이 거래내역 회계 분류해줘. 임차료·급여·복리후생비 위주로.
```
**출력 예시:** 계정과목별 분류 표 + 분류 근거

> **주의:** 분류 결과는 AI 제안이며, 최종 확인은 담당자가 필요합니다.

### 3.6 📋 상대 양식에 우리 자료 채우기

**용도:** 외부 기관이 요청한 양식에 우리 자료 자동 기입  
**입력 예시:**
```
[빈 양식 파일 첨부] + [우리 회사 자료 첨부]
이 입찰 참가 신청서 양식에 우리 회사 자료 채워줘.
```

## 4. 자료 첨부 가이드

### 4.1 지원 형식

| 형식 | 설명 |
|------|------|
| PDF | 텍스트 기반 권장 (스캔본은 OCR 제한) |
| DOCX | Word 문서 |
| XLSX | Excel 스프레드시트 |
| PPTX | PowerPoint (텍스트 추출) |
| TXT / MD | 텍스트 파일 |
| JPG / PNG | 이미지 (텍스트 OCR 제한) |

### 4.2 파일 크기 등급

| 등급 | 크기 | 처리 방식 |
|------|------|-----------|
| S | 50KB 이하 | 즉시 처리 (< 5초) |
| M | 200KB 이하 | 청크 분석 (10~30초) |
| L | 1MB 이하 | 제한 처리 + 진행률 표시 |
| XL | 1MB 초과 | Team Hub 연결 권장 |
| Media-L | 음성·이미지 多 | 텍스트 변환 후 처리 |

> **XL 파일:** Team Hub PC와 페어링하면 제한 없이 처리 가능합니다 (Q3 출시 예정).

## 5. 외부 송신 0 검증

Butler는 모든 처리를 로컬에서 수행합니다. 이를 실시간으로 확인할 수 있습니다.

**Egress 배지 위치:** 화면 우상단 🔒 아이콘

**사용 방법:**
1. 🔒 배지 클릭 → 상세 패널 펼치기
2. 패널에서 확인 가능한 항목:
   - 외부 송신 바이트: 항상 0
   - DNS 요청: 0
   - HTTP/HTTPS 요청: 0
   - 판정: PASS
3. "Download Report" 클릭 → `egress_report_YYYYMMDD.json` 저장
4. JSON 파일로 IT·보안팀에 증적 제출 가능

**Wi-Fi OFF 테스트 절차:**
1. Wi-Fi 완전 차단
2. Butler에서 작업 실행
3. 정상 응답 수신 확인 → 로컬 처리 증명

## 6. 진행률 표시 + 중단

**진행 단계 표시:**
- `분석 시작` → `청크 처리 (N/M)` → `통합` → `검증` → `완료`
- 예상 잔여 시간 실시간 표시

**[중단] 버튼 사용 시점:**
- 응답 방향이 예상과 다를 때
- 처리가 너무 오래 걸릴 때 (기본 타임아웃: 180초)
- 파일 첨부를 잘못했을 때

**부분 결과 활용:**
- 중단 시 "부분 결과 보기" 버튼 표시
- 지금까지 처리된 청크의 결과를 확인 가능
- 결과 JSON 파일로 저장됨
