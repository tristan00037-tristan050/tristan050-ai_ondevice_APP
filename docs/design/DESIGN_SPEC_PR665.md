```
─────────
출처: 국민건강보험공단·국민연금공단·근로복지공단 공식 안내 (2026-05-01 기준)
```

- 상단 우측: **출처 배지** "✓ 검증된 사실" (배경 `#0F7B0F` 10% opacity, 텍스트 `#0F7B0F`, 둥근 모서리 4px)

FactPack 매칭과 동일 구조이되:
- 배지: "✨ AI 생성" (배경 `#10367D` 10% opacity, 텍스트 `#10367D`)

### 처리 중 상태 (엔터 직후)

```
┌─────────────────────────────────────────────┐
│                                             │
│  (입력 비활성, 회색 톤)                       │
```

- 응답 완료(`complete` 이벤트) 또는 정지 시 → 일반 상태 복귀

## 7. 사이드바 — 대화 히스토리 (결함 D 해결)

### 구조

```
┌──────────────────┐
│  + 새 대화        │  ← 클릭 시 메인 영역 초기화 (첫 진입 화면)
│                  │
│  ───────────     │
│                  │
│  오늘             │  ← 그룹 헤더 (#666666, 12px)
│   한국 4대 보험   │  ← 활성: 배경 #10367D 10%, 텍스트 #10367D
│   안녕            │
│   회의록 작성     │
│                  │
│  어제             │
│   ...            │
│                  │
│  지난 7일         │
│   ...            │
│                  │
│  ───────────     │
│  ⚙️ 설정          │
└──────────────────┘
```

### 동작

- 각 대화 항목은 첫 사용자 발화의 앞 20자를 제목으로 자동 생성 (Claude/GPT 패턴)
- 클릭 시 해당 대화의 메시지 전체를 메인 영역에 로드
- 호버 시 우측에 ⋯ 메뉴 표시 (이름 변경 / 삭제)
- 그룹: 오늘 / 어제 / 지난 7일 / 지난 30일 / 더 오래 (Claude.ai 패턴)
- 빈 상태: "아직 대화가 없습니다" 메시지

### 이름 변경 기능 (필수 — 1단계 우선순위)

대화 항목의 자동 생성 제목을 사용자가 원하는 이름으로 변경 가능. Claude.ai/ChatGPT 표준 패턴:

**진입 경로 2가지**:
1. 호버 시 우측 ⋯ 메뉴 → "이름 변경" 클릭
2. 항목 더블클릭 (Claude.ai 단축 패턴)

**편집 모드**:
- 항목 자리에 inline `<input type="text">` 표시 (별도 모달 없음)
- 기존 제목이 input의 기본값으로 채워짐
- input은 자동으로 포커스 + 전체 텍스트 선택 (사용자가 즉시 덮어쓰기 가능)
- input 너비는 항목 영역과 동일, 패딩·폰트도 동일하여 시각 흐름 끊기지 않음
- input 테두리: `1px solid #10367D` (편집 중임을 강조)

**저장 동작**:
- Enter 키 → 저장 + 편집 모드 종료
- Esc 키 → 취소 (변경사항 폐기) + 편집 모드 종료
- input blur (다른 곳 클릭) → 자동 저장 (Claude.ai 패턴)
- 빈 문자열 또는 공백만 입력 시 저장 거부 (기존 제목 유지)
- 최대 길이 100자

**localStorage 갱신**:
- `Conversation.title` 필드만 갱신
- `updated_at` 필드도 함께 갱신 (정렬 영향 없음 — 그룹 분류는 created_at 기준)

**자동 생성 제목과 사용자 지정 제목 구분**:
- `Conversation` 스키마에 `title_is_custom: boolean` 필드 추가
- 새 대화: 자동 생성 (`title_is_custom = false`)
- 사용자 변경 후: `title_is_custom = true`
- 자동 제목인 경우 첫 발화 갱신 시 자동 갱신 가능 (드문 케이스)
- 사용자 제목인 경우 절대 자동 갱신하지 않음

**접근성**:
- input에 `aria-label="대화 이름 변경"` 부여
- 스크린 리더: "이름 변경 모드 진입" 발화

### 삭제 기능 (필수 — 1단계 우선순위)

호버 시 ⋯ 메뉴 → "삭제" 클릭:
- 확인 모달 표시 (Claude.ai 패턴): "대화를 삭제하시겠습니까?" + [취소] [삭제]
- 삭제 버튼 색상: `#B91C1C`
- 확인 시 localStorage에서 해당 conversation 제거
- 현재 보고 있는 대화가 삭제된 경우 첫 진입 화면(빈 상태)으로 자동 전환

### 저장 방식

- **로컬 SQLite** (Tauri 환경) 또는 **브라우저 localStorage** (개발 빠른 진행 우선)
- 본 PR에서는 **localStorage 우선 구현** (베타 발송 일정 보호)
- 정식 베타 도입 후 SQLite로 마이그레이션 (별도 PR)

### 데이터 스키마 (localStorage)

```typescript
interface Conversation {
  id: string;              // UUID
  title: string;           // 첫 발화 앞 20자 (또는 사용자 지정)
  title_is_custom: boolean; // 사용자가 이름 변경했는지 여부
  created_at: string;      // ISO 8601
  updated_at: string;      // ISO 8601
  messages: Message[];
}

interface Message {
  id: string;
  role: "user" | "butler";
  content: string;
  timestamp: string;
  source?: "factpack" | "llm";  // butler role only
  fact_id?: string;             // factpack only
  score?: number;               // factpack only
}
```

키: `butler.conversations` (배열로 저장).

## 12. SSE 이벤트 → UI 매핑

PR #663에서 sidecar의 SSE 이벤트가 정의됨. 이 이벤트들이 UI에 어떻게 반영되는지 정의.

| 이벤트 | UI 동작 |
|---|---|
| `meta {source: "factpack", fact_id, score}` | Butler 발화 컴포넌트 생성 + "✓ 검증된 사실" 배지 표시 |
| `meta {source: "llm"}` | Butler 발화 컴포넌트 생성 + "✨ AI 생성" 배지 표시 |
| `phase_start {phase, total_steps}` | "생각 중 ●●●" 표시 |
| `chunk_progress` | (옵션) 진행률 % 표시 — 첨부 파일 케이스만 |
| `chunk_done` | "생각 중" 유지 (verify_start까지) |
| `reduce_start` | "정리 중..." 표시 |
| `verify_start` | "확인 중..." 표시 |
| `complete {result_text}` | result_text를 Butler 발화 본문으로 즉시 표시 + 입력창 활성화 |
| `error {error_class, message}` | Butler 발화에 에러 메시지 표시 (`#B91C1C`) + 입력창 활성화 |
| `cancelled` | Butler 발화에 "중단됨" 표시 + 입력창 활성화 |
| `heartbeat` | 무시 (단지 연결 유지 신호) |
