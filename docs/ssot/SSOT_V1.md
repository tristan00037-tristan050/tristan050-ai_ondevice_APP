# SSOT v1.0 (정본)
AI 온디바이스 기업용 플랫폼 x – 최종 개발 목표/방향/운영 강제 문서

이 문서는 "말로 하는 계획"이 아니라, 레포/CI/검증 체인에서 강제로 지켜지는 단일 기준(SSOT)입니다.
이 문서와 충돌하는 방향 변경은 PR로만 가능하며, SSOT 변경 시 CHANGELOG+ADR 동반이 강제됩니다.

---

## 0) 제품 정의 (한 문장, 절대 고정)
우리 서비스는 "클로드 코드처럼 회사의 모든 부서·개인 업무를 돕되, 원문은 기기 밖으로 절대 나가지 않으며(온디바이스), 필요 시 고객사가 사내망 내부 서버(옵션)를 붙여 대용량/고부하/인덱싱/정책/감사를 보완하는 기업용 AI 온디바이스 플랫폼"이다.

---

## 1) 제품이 해결하는 문제 (고정)
1. 외부 AI/외부망을 쓰기 어려운 기업(망분리/내부망/감사 강함)에서 "업무 도우미"를 쓰게 한다.
2. 원문(문서/메일/티켓/로그/설계/소스/패치/PR)을 유출하지 않고 업무를 빠르게 한다.
3. 기업 운영 조건(권한/승인/감사/업데이트 검증)을 사람 운영이 아니라 기술로 자동 강제한다.

---

## 2) 시스템 경계 (고정)
### 2-1) 온디바이스 레이어(기본, 항상)
- 실행: PC/브라우저/사내 태블릿/허가된 스마트폰
- 역할: 버틀러(요약/정리/검토/초안/결정 보조)를 기기에서 우선 수행
- 원칙: 원문/원문조각은 기기 밖으로 절대 전송/저장하지 않는다.

### 2-2) 내부 서버(OS Gateway, 옵션)
- 실행: 고객사 내부망(개발=localhost)
- 역할(서버는 AI 코어가 아니다):
  1) 정책/권한/조직(테넌트/역할)
  2) 감사 로그(meta-only)
  3) 사내 시스템 연동(ERP/메일/CS/Drive 등) — 클라이언트 직접 호출 금지
  4) 업데이트/검증/배포(서명/해시/롤백)
  5) (선택) 내부 보조 컴퓨트 — 조건부만, 기본 OFF

---

## 3) 버틀러(전사 업무 도우미) 정의 (고정)
### 3-1) 버틀러 5대 기능
1) 빠른 요약
2) 작업 정리(체크리스트)
3) 검토 보조(문서/설계/코드 변경 위험 포인트)
4) 초안 작성
5) 결정 보조(승인/보류/추가 확인 필요)

### 3-2) 버틀러 출력 규격(하드룰)
출력은 항상 3블록:
- 핵심 포인트
- 결정(승인/보류/추가 확인 필요)
- 다음 행동(사용자가 지금 할 일)

금지:
- 원문 인용/문장 발췌/줄 단위 복사
- 원문 조각을 근거로 포함
근거는 meta-only(상태/카운트/ID/버전/요청ID)만 허용한다.

---

## 4) 절대 불변 규칙(하드룰 12개)
### 4-1) 원문/유출 0
1) 원문/원문조각 외부 전송 0
2) 원문 저장 0 범위는 DB/디스크/이벤트뿐 아니라 로그/예외/오류리포트/리포트/증빙까지 포함
3) 서버/운영에 남기는 것은 meta-only(숫자/상태/ID/버전/요청ID)만

### 4-2) 모드/경계
4) Mock: 네트워크 0(telemetry 포함)
5) Live: 클라이언트→BFF(OS Gateway) 경계 고정, 직접 외부 호출 금지
6) OS 정책 헤더(테넌트/사용자/역할) 누락 즉시 FAIL

### 4-3) 검증/운영 강제
7) verify=판정만(설치/다운로드/네트워크는 preflight에서만)
8) fail-closed(검증 불가/결측/NaN/분모0 등은 통과 금지)
9) 업데이트/적용은 검증 통과 없으면 적용 0(상태 불변)

### 4-4) 반출(Export)
10) Export는 2단계(미리보기→승인)
11) 승인 없으면 반출 0
12) 승인 시 감사(meta-only) 필수

---

## 5) 추가 고정 3개(반드시 봉인, 2026-02-18/02-23 결론)
### 5-1) 업무팩/팀 스위치 우회 즉시 BLOCK
고정 문장:
"업무팩(팀/개인 스위치)은 템플릿/룰/용어 확장만 가능하며, 데이터 이동/반출/정책판단/기록/검증/라우팅은 반드시 공통 코어를 통한다. 우회 경로는 즉시 BLOCK."

봉인 방식(필수 3종):
- SSOT 문장(위 고정 문장)
- 런타임 경계(코어 API 직접 접근 불가)
- verify 스캔(forbidden import/manifest 금지 필드 발견 시 BLOCK)

### 5-2) 원문0 범위에 로그/예외/덤프 포함(전수 강제)
고정 문장:
"원문 저장 0은 DB/디스크/이벤트뿐 아니라 로그/예외/오류리포트/리포트/증빙에도 원문(또는 원문 조각)이 포함되면 즉시 BLOCK이다."

봉인 방식(필수 3종):
- SSOT 문장(위 고정 문장)
- 런타임 logger(meta-only 타입만 허용)
- verify 전수 스캔(docs/ops/**, scripts/** 금지키/긴 라인/배열 덤프 차단)

### 5-3) 상품성 패키지(매출로 이어지는 MVP)
고정 MVP:
"전사 공통 3 시나리오 E2E + 승인 반출 0 + 외부 전송 0 + 운영 콘솔 v0"

---

## 6) MVP(Phase 1) 목표 (고정)
Phase 1에서는 아래가 "제품이 보이는 최소 조건"이다.
1) 3 시나리오 E2E
   - 문서 검색/요약
   - 초안 작성 + 승인 + 반출(승인 없으면 0)
   - 헬프데스크 티켓 요약/다음 단계
2) Export 2단계 + 감사(meta-only)
3) raw0 전수 봉인(로그/예외/리포트/증빙)
4) 업무팩 우회 봉인
5) 성능 수준 추적(accuracy/latency/ram + 최소 1개 P95 예산)

---

## 7) 성능/품질 지표 (고정)
필수 지표 3개:
- accuracy (높을수록 좋음)
- latency_p50_ms / latency_p95_ms (낮을수록 좋음)
- ram_peak_mb (낮을수록 좋음)

P95 예산(최소 1개)은 Phase 1에서 숫자로 SSOT에 고정한다.
예산 초과 시 회귀로 판단하고 차단 또는 degrade한다.

---

## 8) PASS 단일 진실원(말로 PASS 금지)
PASS/개선/종결은 아래가 동시에 있을 때만 인정한다.
- strict 결과(출력 기반)
- eligible 결과(출력 기반)
- CI PASS(출력 기반)
- 지표(meta-only 숫자 증빙)

---

## 9) 방향 변경 규칙(강제)
- SSOT_V1.md 변경 시 CHANGELOG.md + DECISIONS(ADR) 변경이 반드시 동반되어야 한다.
- 이 규칙은 CI/가드 체인에서 자동으로 강제된다.
