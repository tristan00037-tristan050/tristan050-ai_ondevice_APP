## 1. 강점 / 약점 / 기회 / 위협 (SWOT)

### 1-1. 강점 (Strength)

| # | 강점 | 정량 근거 |
|---|---|---|
| S1 | 결함 A 본질 해결 | 정답률 50% → **100%** (4대보험 케이스) |
| S2 | 응답 시간 압도 | 31s → **0.5~2.3 ms** (약 **13,000배** 단축) |
| S3 | 환각 0% | 검증된 답변만 반환, LLM 거치지 않음 |
| S4 | 출처 자동 부착 | 답변 본문에 공식 출처·검증일·유효기간 푸터 |
| S5 | 외부 의존성 최소 | pydantic 외 의존성 0 (온디바이스 우선 원칙 준수) |
| S6 | 한국어 어절 변화 강건 | char 2-gram + light stemming, KoNLPy 의존성 0 |
| S7 | 테스트 커버리지 | 40 PASS (스키마·로딩·매칭·정규화·성능·감사메타·카테고리 라우팅·false-positive 안전성) |
| S8 | False-positive 0건 | 8개 무관 쿼리 전부 미매칭 검증 |
| S9 | Local-only 원칙 100% | 외부 API 호출 0건 |

### 1-2. 약점 (Weakness)

| # | 약점 | 영향도 | 보완 방향 |
|---|---|---|---|
| W1 | Fact 50문항으로는 도메인 커버리지 좁음 | 중 | Fact Pack v2 (Day 15+)에서 200문항 확장 |
| W2 | 큐레이션 대표 직접 의존 (자동화 X) | 중 | 베타 피드백 후 운영팀 큐레이션 SOP 마련 |
| W3 | 회사별 fact pack 추가 UI 미구현 | 낮 | 베타 회사 도입 후 우선순위 결정 |
| W4 | 매처 임계값(0.55) 결정이 50문항 기준 — 200문항 확장 시 재튜닝 필요 | 낮 | v2 확장 시 자동 임계값 탐색 스크립트 도입 |
| W5 | LLM 폴백 응답의 환각은 여전히 차단 못함 | 중 | PR #664~ 후속에서 LLM 응답에도 출처 강제 표기 도입 |
| W6 | GUI 검증은 대표 수동 확인 영역 | 낮 | Day 6 .dmg 재빌드 + 카드 6개 시연 시 검증 |

### 1-3. 기회 (Opportunity)

| # | 기회 | 활용 방안 |
|---|---|---|
| O1 | 한국 규제 산업 (금융·의료·공공·국방)의 클라우드 회피 수요 | 온디바이스 + 한국 사실 큐레이션 → 시장 진입 장벽 |
| O2 | Stanford AI Index 2026의 환각 22~94% 보고로 사실 검증 시장 부각 | 마케팅 자료에 "검증된 사실 답변 100%" 강조 |
| O3 | 글로벌 경쟁자(Glean·CustomGPT) 한국어 부재 | 한국 시장 선점 후 동남아 다국어 확장 |
| O4 | 한국 중소·중견 기업의 좌석당 과금 거부감 | 회사 단위 라이선스로 차별화 |

### 1-4. 위협 (Threat)

| # | 위협 | 가능성 | 시기 | 대응 |
|---|---|---|---|---|
| T1 | OpenAI/Anthropic의 도메인 fact 자체 학습으로 LLM 정확도 향상 | 높음 | 2026 하반기 | 결정론·온디바이스·출처표기·응답속도 차별화 유지 |
| T2 | 한국 스타트업의 유사 시도 | 중 | 2026 하반기 | 베타 5→30곳 lock-in으로 선점 |
| T3 | Microsoft Copilot 그라운딩 강화 | 높음 | 진행 중 | 클라우드 불가 영역(국방·보안) 집중 |
| T4 | Glean의 한국어 강화 | 중 | 2027~ | Fact 200→1000 확장 + 회사별 확장 기능으로 깊이 차별화 |
| T5 | 베타 회사에서 fact 답변 오류 보고 시 신뢰 추락 | 중 | 즉시 가능 | 모든 fact 출처 직접 확인 + 베타 첫 주에 매일 매칭 로그 검토 |
